# Voice Profile Guide

ボイスプロファイル（文体マッチング）の作成・活用ガイド。

## 概要

LLM にユーザーの文体を学習させ、一貫した voice でコンテンツを生成する手法。
ただし、LLM の文体学習には構造的な制限がある（後述）。

## ボイスプロファイルの作り方

### Step 1: サンプル収集

自分の文章を 5-10 本集める。以下の多様性を持たせる:

- **フォーマル vs カジュアル**: ブログ記事、Slack メッセージ、技術ドキュメント
- **長文 vs 短文**: 論考、ツイート、コメント
- **異なるトピック**: 技術、日常、意見表明

### Step 2: パターン抽出

サンプルを Claude に渡し、以下を抽出させる:

```
あなたは文体分析の専門家です。以下のテキストサンプルから、著者の文体パターンを抽出してください:

1. **語彙の傾向**: よく使う単語・表現、避けている表現
2. **文構造**: 文の長さ、接続詞の使い方、段落の区切り方
3. **トーン**: フォーマル度、ユーモアの有無、断定度
4. **修辞的特徴**: 比喩、疑問形、リスト好き、など
5. **避けていること**: 使わない表現パターン

[サンプルテキストを貼り付け]
```

### Step 3: プロファイル化

抽出結果を構造化する:

```markdown
## Voice Profile: {名前}

### 特徴
- 語彙: {特徴}
- 文構造: {特徴}
- トーン: {特徴}

### DO
- {この人らしい表現パターン}

### DON'T
- {この人が絶対に使わない表現}

### サンプル
> {最も「らしい」一文}
```

### Step 4: CLAUDE.md に追記

プロファイルを CLAUDE.md またはスキルの設定に追記する。

## 活用方法

### /obsidian-content での活用

`/obsidian-content` スキルで Vault のコンテンツを生成する際、Voice Profile を参照させる:

```
/obsidian-content ニュースレター書いて（Voice Profile に従って）
```

### /rewrite での活用

`/rewrite for [自分の名前]` でカスタムペルソナとして自分の voice を指定可能。

## 制限事項

### LLM の文体学習の構造的問題

1. **セッション跨ぎの記憶なし**: Voice Profile は毎セッションで再読み込みが必要。CLAUDE.md に書くか、スキルの冒頭で参照する
2. **表層的な模倣**: LLM は語彙・文構造のパターンは模倣できるが、思考の癖や独自の視点までは再現できない
3. **長文での劣化**: 短いテキスト（ツイート、Slack）では効果的だが、長文になるほど「LLM っぽさ」が漏れ出る
4. **自己評価バイアス**: LLM に「この文体は合っていますか？」と聞いても、自分の出力を高く評価する傾向がある（Gemini 指摘）

### 推奨する使い方

- **短文生成に使う**: Slack メッセージ、ツイート、メール返信
- **下書きとして使う**: LLM 出力をそのまま使わず、自分で手を入れる前提
- **定期的にプロファイルを更新する**: 文体は時間とともに変化する

## 撤退条件

3回使って効果を実感できなければ、このガイドを削除する。
Voice Profile のメンテナンスコストが効果を上回る場合も同様。
