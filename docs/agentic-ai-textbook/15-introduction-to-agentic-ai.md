# 第15章 Introduction to Agentic AI（エージェント型AI入門）

> **一文サマリ:** エージェント型AIとは、LLMが「観測 → 推論 → 行動」をループで回し、目標達成かユーザーへの問い合わせまで自走するシステム。本章はその全体地図で、Ch16以降の各章が積み上げる「層」を先に俯瞰する。
> **PDF参照:** §15, p.292-294

これは Ch16 以降と違って「作る概念」ではなく**地図**の章。6フィールド型は次章から始まる。ここでは「なぜエージェントが要るのか」と「これから何を積み上げるのか」を掴む。

---

## 15.1 チャットボットからエージェントへ

単発チャットボットは「1メッセージ受けて1応答を返して待つ」。エージェントは違う。**LLMをループの中で動かす**:

1. 環境から観測を受ける（ユーザーのメッセージ、ツールの出力、センサーデータ）
2. 次に何をするか推論する
3. 行動する（ツール呼び出し、コード実行、APIリクエスト）
4. 目標達成、または「人間に聞く」まで 1→3 を繰り返す

この「待つ」と「自走する」の差が、本書全体の出発点。

## 15.2 エージェント化で生まれる5つの根本課題

単発のモデル呼び出しでは解けない課題が5つ生まれる。これがそのまま後続章のテーマになる。

| 課題 | 中身 | 担当章 |
|---|---|---|
| **Persistence（永続）** | 何をやり、何が失敗し、どんな文脈を作ったか — ターン・セッション・日をまたいで覚える | Ch17 Memory |
| **Grounding（接地）** | 学習データに無い最新・ドメイン固有の知識へアクセスする | Ch16 RAG |
| **Action（行動）** | DB・API・ファイル・ブラウザに、定義されたインターフェース越しに触る | Ch21 MCP |
| **Coordination（協調）** | 1エージェントの手に余るタスクを、複数の専門エージェントで分担・委譲・交渉する | Ch23 A2A / Ch24 Multi-Agent |
| **Safety（安全）** | 自走する以上、ガードレール・人間の監督・不確実時の安全な縮退が要る | Ch18 Harness / Ch26 UI |

## 15.3 レイヤードアーキテクチャ（本書の骨格）

プロダクションのエージェントは**層**で組む。各層が1つの問題を解き、Ch16-26 が下から上まで全スタックを埋める。下の対応表が、この教科書を読み通したとき手元に積み上がるものの全体像。

| 層 | 章 | 役割 | リサーチエージェントでの役割 |
|---|---|---|---|
| 知識層 | Ch16 RAG | クエリ時に関連文書を引いて接地 | 収集済みの記事・論文コーパスから関連箇所を引く |
| 永続層 | Ch17 Memory | 相互作用をまたいで記憶 | 「この情報源は当たり/外れ」の学習、過去の調査結果の蓄積 |
| 実行時層 | Ch18 Harness | ループ・文脈予算・ツール発行・エラー回復・観測性 | エージェント本体を回す「OS」。文脈溢れと無限ループを防ぐ |
| アーキ層 | Ch19 Design Patterns | ReAct / plan-execute / reflection 等の構造 | 「探す→読む→辿る→まとめる」の制御戦略を選ぶ |
| 評価層 | Ch20 Environments | エージェント挙動を測る・ベンチ | 収集の質・軌跡の良し悪しを評価する |
| ツール統合 | Ch21 MCP | ツール発見・呼び出しの標準（ハードのUSB相当） | 検索API・スクレイパ・要約器を標準I/Fで繋ぐ |
| 能力層 | Ch22 Skills | 基本ツールを超えた能力の獲得・合成 | 「論文を構造化要約する」等の再利用可能スキル |
| エージェント間 | Ch23 A2A | 異種エージェント間の発見・委譲・集約 | 外部の専門エージェントに下請けを出す |
| 協調層 | Ch24 Multi-Agent | 階層委譲・P2P交渉・議論・群知能 | 収集役 / 選別役 / 要約役を分業させる |
| 実装層 | Ch25 Frameworks | 上記を実装するツールキット（LangGraph 等） | 実際に組むときの土台選び |
| 対話層 | Ch26 Agentic UI | ストリーミング・承認フロー・信頼を作るUX | 採否の人間レビュー、進捗の可視化 |

> **+ 上級・任意（Ch12）:** ここまでは「既存モデルの上で組む」。Ch12 は一歩進んで、エージェントの軌跡そのものをRL（STaR / Reflexion / RLEF 等）で学習させる話。Tier 1 の最後に置く。

## 15.4 全体の統合 — 図15.1 を言葉で

各層は孤立せず、密に噛み合った1つのシステムになる。中心に**Agent Core**（Parts II-III の推論力を持つLLM）が座り、`perceive → reason → act` ループを回す。

```
            ┌─────────────────────────────────────────────┐
            │  User / Human-in-the-Loop      [UI: Ch26]    │
            └──────────────▲──┬───────────────────────────┘
                  feedback  │  │ goals
            ┌──────────────┴──▼───────────────────────────┐
            │  Harness & Orchestration            [Ch18]   │
            │  Context Mgmt / State・Loop / Guardrails / Obs│
            └──────────────▲──┬───────────────────────────┘
                           │  │
            ┌──────────────┴──▼───────────────────────────┐
            │  Agent Core (LLM)          ┌─ Agentic Loop ─┐│
            │   Perceive → Reason/Plan → Act ──┐ [Ch19]  ││
            │      ▲──────── feedback / observe ┘         ││
            └────┬────────┬─────────┬──────────┬──────────┘
                 ▼        ▼         ▼          ▼
            ┌────────┐┌────────┐┌────────┐┌────────────┐
            │  RAG   ││ Memory ││ Tools  ││Other Agents│
            │ [Ch16] ││ [Ch17] ││(MCP/21)││  (A2A/23)  │
            └───┬────┘└───┬────┘└───┬────┘└─────┬──────┘
                └─────────┴─────────┴───────────┘
            ┌─────────────────────────────────────────────┐
            │  Environment (web, code, DB, 実世界) [Ch20]  │
            └─────────────────────────────────────────────┘
```

繋がりの要点:

- **Agent Core** が中心で `perceive → reason → act` を実行する。
- **RAG** が各推論ステップの前に関連知識を流し込み、**Memory** がステップ・セッションをまたいだ継続性を与える。
- **Harness** が全部を統括する — いつ検索するか、いつツールを呼ぶか、いつサブエージェントに委譲するか、いつ人間に聞くかを決める。
- **MCP** が全外部ツールへの標準I/F、**A2A** がエージェント間通信の標準I/F。
- **Design Patterns** が高レベル戦略（ReAct / plan-execute / reflection）を定め、**Frameworks** がその具体実装を与える。
- **UI層** が人間へループを閉じる — 監督・修正・協働のために。

全章を貫く視点: エージェント型AIは「プロンプトの工夫」ではない。**文脈管理・エラーハンドリング・安全ガードレール・観測性を各層で工学する**仕事だ。

## この章のまとめ

- エージェント = LLM を `observe → reason → act` のループで回し、目標まで自走させる仕組み。単発チャットボットとの差はここ。
- 5つの根本課題（Persistence / Grounding / Action / Coordination / Safety）が、後続章のテーマに直結する。
- 本書は層で積む: 知識(RAG) → 永続(Memory) → 実行時(Harness) → アーキ(Patterns) → 評価 → ツール(MCP) → 能力(Skills) → 協調(A2A/Multi-Agent) → 実装(Frameworks) → 対話(UI)。
- 例題の軸は終始リサーチエージェント。各層がその設計図の一部になる。
