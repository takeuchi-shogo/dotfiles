01アーキテクチャ全体図

harness は 5 レイヤーの積層構造。上の層が「何をすべきか」を定義し、下の層が「実際に守らせ・覚え・改善する」。クリックで各レイヤーの詳細へ。

L1 指示層 CLAUDE.md (~130行, Progressive Disclosure) / references/ (~120 docs) / rules/ (言語別 + 委譲ルール) → L2 強制層 hooks (9 events / 60 commands) / Rust binary tools/claude-hooks (8 modules) / Lefthook / settings.json deny rules → L3 能力層 skills 121 / agents 23 (+archived 10) / plugins 16 → L4 マルチモデル層 Fable=指揮 / Opus=推論 / Sonnet=実装 / Codex Review Gate / Gemini 1M / cmux Workers → L5 記憶・自己改善層 7層メモリモデル / patterns.jsonl → auto-triage → promote-learnings / nightly jobs / Obsidian 単方向同期 →

L1指示層 — Instruction Layer

「Claude に何をどう判断させるか」を宣言する層。CLAUDE.md は約 130 行に絞り、<important if> 条件タグと references/ への Progressive Disclosure で詳細を遅延展開する。指示は常時コンテキストに乗るためトークン予算と注意 (attention) を直接消費する — だから「少なく・濃く・条件付き」が原則。

コンポーネントパス役割
CLAUDE.md.config/claude/CLAUDE.md~130行のグローバル指示。Foundation / Delegation / 設計原則 / 条件付き important ブロック
references/.config/claude/references/~120 docs。決定表・モデルルーティング・harness 安定性などの詳細仕様を必要時のみ展開
rules/.config/claude/rules/言語別ルール (Go/TS) + Codex/Gemini 委譲ルール
PLANS.mdrepo root非自明な変更のプランニング規約。plansDirectory は tmp/plans/
agent-harness-contract.mddocs/harness 変更時の契約 (何を壊してはいけないか)
原則 ドキュメント＝インフラ（仕様書は耐荷重構造物。2 回説明したら書き下ろす）/ IFScale: 指示の「数」が品質を決める — 増やすより削って濃くする

L2強制層 — Enforcement Layer

「プロンプトに書いたお願い」は確率的にしか守られない。static-checkable なルールはすべてこの層に降ろして決定論的に強制する。hooks は 9 イベント / 60 コマンドで Claude のツール実行をリアルタイムに監査・ブロックし、ホットパス（毎 Bash・毎 Edit に走るガード）は Rust binary に統合して latency を抑えている。

コンポーネントパス役割
hookssettings.json → scripts/{policy,runtime,lifecycle,learner}/9 events / 60 commands。completion-gate, prompt-injection-detector, file-proliferation-guard など
Rust binarytools/claude-hooks8 modules (events / io / main / post_any / post_bash / post_edit / pre_tool / user_prompt)。高頻度 hook を 1 バイナリに統合
Lefthooklefthook.ymlポリグロット pre-commit + commit-msg 検証
deny rulessettings.jsongit commit --no-verify 禁止、lint config 保護などの拒否リスト
原則 Determinism boundary — linter / ast-grep / hook / test で表現できるルールはプロンプトに書かない。「指示」は壊れるが「機構」は壊れない

L3能力層 — Capability Layer

「スキル＝知識ベース、エージェント＝実行コンテキスト」。skill は手順・ドメイン知識を on-demand で注入し、agent は隔離されたコンテキストで実行を担う。121 skills のうち外部由来 (google/skills 13, mizchi/skills 18 ほか) は hash 検証付きで vendoring。使わない skill は skillOverrides で抑制し、listing budget（注意の押し出し）を管理する。

コンポーネントパス役割
skills (121).config/claude/skills/ワークフロー / 知識管理 / レビュー / UI / 外部 vendored。検索は下のカタログへ
agents (23 + 10 archived).config/claude/agents/レビュー系が最厚 (12)。30 日評価後に退役 → archived へ (Build to Delete)
plugins (16)settings.json 駆動superpowers / codex / pr-review-toolkit など。task claude:plugins で別マシン再現
skills-lockskills-lock.json外部 skill の出自・hash 固定。drift は task skills:verify で検出
原則 Build to Delete — ハーネスは過渡的技術。設計時に「何が改善されればこれは不要か」を問う / 削除は 30 日評価後 (harness-stability)

L4マルチモデル層 — Multi-Model Layer

単一モデルに全部やらせない。メイン (Fable) は判断・統合に限定し、実装・探索は Sonnet サブエージェントへ並列委譲、深い推論サブタスクは Opus、レビューは外部モデル (Codex) でバイアスを相殺する。協調プロトコルの設計が品質差異の 44% を占め、モデル選択は ~14% — だから Scaffolding に投資する。

Tierモデル役割
指揮Fable (メイン)判断・統合・最深推論。実装はしない
推論OpusPlan 草案・設計分析・根因調査
実装Sonnet実装・探索・テスト。並列 Agent 実行
批評Codex (gpt-5.5)Review Gate (S規模以上)・Spec/Plan Gate (M規模以上)。外部視点でバイアス相殺
分析Gemini1M コンテキスト分析・リサーチ・マルチモーダル (PDF/動画/音声)
補助Cursorマルチモデル・Cloud Agent

対話的なセカンドオピニオン（improve / debate / absorb）は cmux Worker (launch-worker.sh) で実モデルとラリーする。サブエージェント一発投げに逃げない、が運用ルール。

原則 Scaffolding > Model — 協調プロトコル選択が品質の支配項。既知バイアス: Claude=萎縮+Sycophancy / Gemini=過度に楽観 / Codex=狭く深い

L5記憶・自己改善層 — Memory & Self-Improvement Layer

セッションを跨いで「学んだこと」を残し、運用ログから改善を半自動で昇格させる層。Claude Code 内蔵の 7 層メモリ（Tool Result Storage → Microcompaction → Session Memory → Full Compaction → Auto Memory Extraction → Dreaming → Cross-Agent Communication）のうち、harness が介入できるのは L4 以降 — PreCompact hook での状態保存、MEMORY.md / memory/ の構造管理、Agent 定義によるコスト制御がその介入点。

03設計原則

harness 全体を貫く 6 つの原則。

Humans steer, agents execute人間は方向を決め、エージェントが実行する。メインモデルも「指揮」に徹し実装はサブエージェントへ。
Build to Deletehook / script / agent は過渡的技術。設計時に「何が改善されればこれは不要か」を問い、削除コストを最小化する。
Determinism boundarystatic-checkable rules は linter / hook へ降ろす。プロンプトのお願いは確率的、機構は決定論的。
契約層 strict / 実装層 regenerableAPI・型・schema は厳密に定義し、実装は再生成可能に保つ。壊れたら作り直せる側に複雑さを寄せる。
境界で Fail Fast / 内部は Trust暗黙フォールバック・モック残置・NO-OP は絶対禁止。境界で即死させ、内部は信頼して薄く保つ。
Scaffolding > Model協調プロトコル選択が品質差異の 44%、モデル選択は ~14%。診断に使えない信号は実質ゼロ — 観測可能にする。
