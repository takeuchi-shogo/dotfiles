## Source Summary

**Hyperagents** (Zhang, Zhao, Yang, Foerster, Clune, Jiang, Devlin, Shavrina, 2026)

DGM-Hyperagents (DGM-H): タスクエージェントとメタエージェントを単一の編集可能プログラムに統合した自己参照的エージェント。
メタ認知的自己修正により、タスク解決能力だけでなく改善メカニズム自体を改善する。

### 主要手法

1. **メタ認知的自己修正**: 改善プロセス自体が編集対象。改善 → 改善方法の改善 → ...
2. **アーカイブベースのオープンエンド探索**: 成功バリアントの集団を保持、確率的親選択で分岐
3. **永続メモリ**: 因果仮説、合成的洞察、前方計画、過修正診断を蓄積（単なるスコアログではない）
4. **パフォーマンストラッキング**: 世代間メトリクス比較、移動平均トレンド、ドメイン別分離
5. **クロスドメイン転移**: メタレベル改善（メモリ導入、トラッキング）がドメイン横断で転移
6. **複利的自己改善**: 一設定での改善が別設定での継続改善と複合
7. **安全制約**: サンドボックス + リソース制限 + 事前定義評価 + 人間監視

### 結果

- コーディング: 0.140 → 0.340（DGM と同等、ドメイン固有設計なしで達成）
- 論文レビュー: 0.0 → 0.710（静的ベースライン 0.630 を超越）
- ロボティクス報酬設計: 0.060 → 0.372
- 数学採点: 転移ハイパーエージェントで imp@50 = 0.630
- 複利: DGM-H + transfer + ProofAutoGrader = 0.700（ProofAutoGrader 単体 0.670 を超越）

### 実践的知見

- PerformanceTracker クラスが自律的に発生（移動平均、ドメイン別統計）
- 永続メモリが「ベスト世代分析」「過修正検出」「次の戦略提案」を蓄積
- メタ改善の転移: 論文レビュー+ロボティクスで学んだ改善戦略が数学採点でも有効
- 表面的なプロンプト微調整 → 構造化された意思決定パイプライン（チェックリスト、ルール）への進化
