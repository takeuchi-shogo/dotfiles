# Knowledge Pyramid — 4層昇格パイプライン

> Spark 記事の知識ピラミッド概念を AutoEvolve の知識管理に統合する。
> 既存の5レベル蒸留パイプライン（L0-L4）とは直交する品質軸として機能する。

## 概要

知識ピラミッドは、学習データの**エビデンス品質**を4層で分類する。
蒸留パイプライン（L0-L4）が「どこに保存するか」を定義するのに対し、
ピラミッドは「どれだけ信頼できるか」を定義する。

## 4層定義

| Tier | 名前 | 説明 | スコア範囲 |
|------|------|------|-----------|
| 0 | Raw | 未検証の観察・単発イベント | 0.0 - 0.3 |
| 1 | Exploratory | 複数回の観察で仮説が形成された段階 | 0.3 - 0.6 |
| 2 | Benchmark | 定量データで裏付けされた知見 | 0.6 - 0.8 |
| 3 | Doctrine | 複数コンテキストで検証済み、原則として確立 | 0.8 - 1.0 |

## 昇格条件

| 遷移 | 条件 |
|------|------|
| Tier 0 → 1 | 同一パターンが 3 セッション以上で観察 |
| Tier 1 → 2 | 定量メトリクスで効果を確認（A/B delta > +2pp または改善率 > 20%） |
| Tier 2 → 3 | 3 つ以上の異なるコンテキスト（プロジェクト・タスク種別）で有効性を確認 |

## 降格条件

| 遷移 | 条件 |
|------|------|
| Tier 3 → 2 | 新たなコンテキストで反例が確認された |
| Tier 2 → 1 | 定量メトリクスが keep 条件を満たさなくなった |
| Tier 1 → 0 | 60 日間再観察なし |

## 既存パイプラインとの関係

```
蒸留パイプライン（保存先）    知識ピラミッド（品質）
L0 Raw events              × Tier 0-3
L1 Recovery tips           × Tier 0-3
L2 Error fix guides        × Tier 1-3（通常）
L3 Rules                   × Tier 2-3（推奨）
L4 Golden principles       × Tier 3（必須）
```

- L0-L4 は「どこに保存するか」（形式・粒度）
- Tier 0-3 は「どれだけ信頼できるか」（エビデンス品質）
- 両軸は独立: L2 のエントリが Tier 1 であることも、Tier 3 であることもあり得る
- L3 以上への昇格には Tier 2 以上を推奨、L4 には Tier 3 を必須とする

## 運用

- `append_to_learnings()` のデフォルト: `tier: "raw"`, `score: 0.0`
- 呼び出し元が明示的に指定した場合はそちらを優先
- Garden フェーズで tier と score を再評価し、昇格・降格を判定
