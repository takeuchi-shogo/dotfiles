# Verbalized Sampling (VS) ガイド

> 出典: Zhang, Yu, Shi et al. 2025 "Verbalized Sampling: How to Mitigate Mode Collapse and Unlock LLM Diversity" (arXiv:2510.01171)

LLM に回答の**分布**を言語化させることで、RLHF 由来の mode collapse（出力の画一化）をプロンプトのみで緩和する手法。

## 背景: Typicality Bias

RLHF の選好データには **typicality bias** がある — 人間は内容が同等でも「典型的な」回答を好む傾向がある（α ≈ 0.57-0.65, p < 10⁻¹⁴）。このバイアスがアライメント後の出力分布を鋭化させ、多様性を失わせる。

重要な帰結: **アライメント済みモデルは潜在的な多様性を保持している**。適切なプロンプトでそれを引き出せる。

## 3 バリアント

### VS-Standard

モデルに回答の分布を直接言語化させる。

```
この質問に対して、異なる視点・アプローチ・スタイルで回答するとしたら、
どのような分布になるか考えてください。
まず考えられる回答のバリエーションを列挙し、
その中から1つを選んで詳細に回答してください。
```

**適用場面**: 創作、設計案、リファクタリング方針

### VS-CoT

推論ステップを挟んでから分布を生成する。

```
この問題について、まず異なるアプローチの長所・短所を推論してください。
次に、それぞれのアプローチがどの程度適切かの分布を考え、
その分布に基づいて1つのアプローチを選択・実行してください。
```

**適用場面**: 技術選定、アーキテクチャ決定、トレードオフ分析

### VS-Multi

マルチターンで段階的に多様な回答を収集する。

```
Turn 1: 最も典型的なアプローチで回答してください。
Turn 2: 前の回答とは根本的に異なるアプローチで回答してください。
Turn 3: 上記2つのどちらでもない第三のアプローチで回答してください。
```

**適用場面**: best-of-n の候補生成、ブレインストーミング、`/debate`

## 使い分け

| 状況 | 推奨バリアント | 理由 |
|------|---------------|------|
| 単一セッションで多様な案が欲しい | VS-Standard | 最もシンプル。1ターンで完結 |
| 根拠付きの比較が必要 | VS-CoT | 推論過程で各案の強度が明確になる |
| best-of-n の候補生成 | VS-Multi | worktree ごとに異なるターンの回答を使える |
| レビューの多角的検討 | VS-Standard | レビューアーのプロンプトに仕込む |

## 特性

- **temperature / top-p / min-p と直交**: 併用可能。サンプリングパラメータとは異なる軸で多様性を生む
- **安全性に影響なし**: 有害プロンプトへの拒否率 >97% を維持
- **大きいモデルほど効果大**: 1.5-2x の多様性改善（小モデルでは限定的）
- **事実確認タスクには不要**: 正解が1つのタスクでは多様性は不要

## 不採用候補の活用

SSD 研究の知見: 選ばれなかったサンプルにも学習価値がある。

VS-Multi や `/debate` で生成した候補のうち、採用されなかった案も session-trace に記録する:
- 不採用理由を 1 行で添える（「テスト不合格」「パフォーマンス劣後」「スコープ外」等）
- contrastive-trace-analyzer の入力として、採用/不採用の対比分析に使用可能
- 特に難易度の高いタスクでは、不採用案の部分的アプローチが後続タスクで再利用できることがある

> 出典: "Embarrassingly Simple Self-Distillation Improves Code Generation" (arXiv:2604.01193) — 未検証サンプルからの学習が難問で特に有効

## Anti-Patterns

| NG | 理由 |
|----|------|
| 全てのプロンプトに VS を適用 | 事実確認・決定的タスクでは無駄なオーバーヘッド |
| VS を temperature の代替として使う | 直交する手法。併用が正しい |
| VS-Multi で N>5 ターン | 後半のターンは無理に差別化しようとして品質が低下する |
