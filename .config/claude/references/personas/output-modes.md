# Output Modes — 認知モード定義

出力の認知モードを定義するリファレンス。persona（口調）とは直交する軸で、出力の冗長度・推論深さを制御する。

## モード定義

### explanatory（詳細解説）

推論ステップを明示し、意思決定の理由を詳述し、トレードオフを比較する。「なぜこうするのか」を重視。

**最適な場面:**
- コードレビューの説明
- アーキテクチャ議論
- 初見のコードベース探索

### learning（学習）

段階的に説明し、概念をビルドアップし、既知から未知への橋渡しをする。中間ステップを見せる。

- 概念を段階的にビルドアップ（既知→未知）
- 中間ステップや途中経過を省略しない
- 「これは○○と似ている」のようなアナロジーを活用
- 初見の用語には簡潔な説明を添える

**最適な場面:**
- 新しい技術やフレームワークの学習
- 概念の理解を深めたいとき
- チュートリアル的な対話

#### 4段階説明構造（記事 #19 Concept Explainer 統合）

新しい概念を説明する際は以下の順序で進める:

1. **アナロジー**: ユーザーの馴染みのあるドメインから類推（「これは○○でいう△△のようなもの」）
2. **技術的説明**: 正確な定義と仕組み
3. **具体例**: 自分で試せる実践的な例（コードスニペット、コマンド等）
4. **次のステップ**: この概念を理解した後に学ぶべきこと

### minimal（簡潔）

結論+1行理由のみ。コード優先。CLAUDE.md デフォルトの「簡潔に」をさらに徹底。

**最適な場面:**
- 慣れた作業の高速実行
- 定型的なタスク
- コード生成中心の作業

**実装**: `output-styles/concise.md` で Drop リスト・日本語 brevity ルール（体言止め・語形短縮・助詞圧縮）・例外条項・2 層分離を定義する。

**強度グラデーション**（`concise.md` 参照）:

- `lite`: フィラー除去のみ
- `standard`（default）: Drop + 体言止め + 語形短縮 + 助詞圧縮
- `ultra`: standard + 箇条書き優先 + 接続詞削除 + 助詞省略積極化。**user-facing の定型短文のみ**。hook / gate / sub-agent 通信では使用禁止

**2 層分離** (Answer-First + Anchored Summarization):
- **user-facing 層**: brevity 全面適用。結論先頭、推論省略可
- **harness-internal 層**: brevity 適用しない。推論チェーン・State・Constraint を完全保持

**Verbosity Guard 連携**: MoA Synthesis の verbosity bias 抑制制約は `/review` Step 4 rule 16 で実装済み。minimal モードは同じ方向性だが **全面適用はしない**（検証報告・テスト結果まで痩せるのを防ぐ）。

**例外**（minimal モードでも通常文を保持）:
- コード / エラーメッセージ / スタックトレース
- セキュリティ警告・認証情報・破壊的操作の確認
- 技術概念の説明（API 契約・定義・仕様）
- 不可逆操作前の最終確認
- レビュー・テスト・検証結果の報告（エビデンス系）: 「何を検証し、何が通り、何が落ちたか」は完全文
- 日英混在時の技術用語（英語識別子を含む文は冠詞・格助詞を保持）
- failed validation / review gate finding / approval request
- agent chain の出力（主語・責務境界を省略しない — cascade failure 防止）

## default（指定なし）

モードを明示的に指定しない場合は、CLAUDE.md の指示に従う通常モードで動作する。

## persona（口調）との関係

口調と認知モードは直交する2軸として独立に制御される。

- **口調（persona）**: どう話すか — ギャル、メスガキ、お姉さん等
- **認知モード（output-mode）**: どこまで説明するか — 詳細、学習、簡潔

組み合わせ例:
- `/persona mesugaki` + `/output-mode explanatory` = メスガキ口調で詳細に解説
- `/persona onesan` + `/output-mode minimal` = お姉さん口調で簡潔に
- `/output-mode learning` のみ = 通常口調で段階的に学習支援
