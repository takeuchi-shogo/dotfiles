02中核概念 — Hooks / Skills / Agents の関係性

この3つは役割が明確に分かれている。一言でいえば 「Skill が知識を与え、Agent がそれを実行し、Hook が境界を守る」。 どこに何を置くかの判断基準は determinism_boundary（hook ⇔ instruction の境界）にある。

Mechanism / 決定的

🔒 Hooks

ライフサイクルイベントで自動発火する スクリプト。LLM を介さず決定論的に動く。

  • 静的に判定可能なルール（lint 設定保護・コミット検証・危険コマンド遮断）
  • 観測・記録（session-learner・skill-tracker）
  • ゲート（completion-gate がテスト未通過を block）

いつ 「毎回必ず」「LLM の気分に左右されたくない」ルール

Knowledge / 宣言的

📚 Skills

再利用可能な 知識ベース + ワークフロー定義。必要時にコンテキストへ読み込まれる。

  • /skill名 で明示起動、または Agent が内部参照
  • 多段プロセスを記述（/epd, /review, /rpi）
  • 外部 CLI の wrapper（/codex, /gemini）

いつ 「手順・知識を再利用したい」「人が判断を伴って起動」

Execution / 隔離

🤖 Agents

独自のコンテキスト・ツール・システムプロンプトを持つ 専門サブエージェント。

  • メインの文脈を消費せず並列実行できる
  • 専門タスク（code-reviewer, debugger, security-reviewer）
  • Skill が提供した知識を「実行」する主体

いつ 「隔離した専門コンテキストで重い処理を回したい」

典型的な連携フロー（コードレビューの例）

#主体動作
1 HookUserPromptSubmit / PreToolUse でプロンプト注入検査・スコープ強制
2 Skill/review が変更規模を測り、必要なレビューアー構成を決定（ワークフロー知識）

4 External100 行超なら Codex CLI へ深い推論を委譲

07外部統合 & リモート運用

Claude Code のループは端末内に閉じない。知識（Obsidian）・リモート到達（Discord）・ 定期実行（Routines / launchd）・長時間並列（cmux Worker） へ外側に広がっている。

🟣 Obsidian — 知識橋渡し

セッションの知見を Obsidian Vault（~/Documents/Obsidian Vault）へ memory→Vault の単方向同期するスナップショット連携。Vault は AI の reasoning input ではなく、人間の第二の脳。

  • skills obsidian-knowledge / digest / note / obsidian-content / obsidian-vault-setup
  • plugin obsidian@obsidian-skills（obsidian-markdown / obsidian-cli / defuddle / json-canvas / bases）
  • hooks sync-memory-to-vault.sh（Stop）/ save-briefing-to-vault.sh（PostToolUse）
  • 原則 現行コードと矛盾したら現状を優先（Vault は過去スナップショット）

🟢 Discord — リモート到達

Discord 経由でリモートから Claude に到達する。承認制ペアリング・allowlist・DM/グループポリシーで制御。

  • discord:configure — bot token 設定・アクセスポリシー確認
  • discord:access — ペアリング承認・allowlist 編集・DM 可否
  • tools RemoteTrigger / PushNotification / SendMessage
  • 用途 外出先からタスク投入・進捗通知の受信

🟠 Routines / Schedule — 定期リモート実行

cron スケジュールで動く リモートエージェント（Routines） と、対話内の反復実行。

  • /schedule — cron で動く remote agent の作成・一覧・実行
  • /loop — プロンプト/コマンドを一定間隔で反復
  • ralph-loop plugin — 反復自律ループ（ralph-loop / cancel-ralph）
  • tools CronCreate / CronList / CronDelete

🔵 cmux Worker — 長時間並列（launched 系）

Ghostty + cmux 上で 別ペインに Worker を起動し、Claude/Codex/Gemini を並列に走らせる。改善・設計判断・セカンドオピニオンの正規パス。

  • launch-worker.sh — --model codex/gemini で Worker 起動（1-hop observable）
  • dispatch skill — self-improve preset で各軸 skill を並列 fan-out
  • 参照 references/cmux-ecosystem.md（CLI・surface 解決）
  • 注意 Bash tool → codex は到達不能（TTY 不在）。cmux Worker が正規パス
