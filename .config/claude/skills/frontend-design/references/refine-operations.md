# Refine Operations Reference

Impeccable (pbakaus/impeccable) の設計コマンド体系を参考に、5操作に集約したリファレンス。

## distill — 削ぎ落とし

目的: 過剰なUIを本質に戻す。AI生成コードの「とりあえず全部入れ」を解毒する。

チェックリスト:
- [ ] 使われていない装飾要素（グラデーション、シャドウ、ボーダー）を除去
- [ ] ネストが深すぎるコンポーネント階層をフラット化
- [ ] 1画面に複数の視覚的主張がないか（1セクション1メッセージ）
- [ ] カードの中にカードがないか
- [ ] 「見せるため」だけの要素を除去（意味のないアイコン、装飾テキスト）
- [ ] normalize: デザイントークン（色、間隔、フォントサイズ）を CSS 変数に統一

## polish — 仕上げ

目的: 出荷前の最終調整。機能は完成しているが細部が雑なUIを磨く。

チェックリスト:
- [ ] typeset: フォント選定の見直し（generic → distinctive）、サイズ階層の一貫性
- [ ] arrange: 余白の不均一を修正、垂直リズムの統一
- [ ] アライメントのズレを修正（baseline, center）
- [ ] ホバー・フォーカス状態の欠落を補完
- [ ] レスポンシブでの崩れを確認・修正
- [ ] ダークモード対応の一貫性（ある場合）

## critique — 診断

目的: コードを変えずに問題点と改善案を報告する。Read-only 操作。

出力フォーマット:
```
### 診断結果

**総合評価**: [A/B/C/D/F]

#### 問題点
1. [severity: high/medium/low] 問題の説明 → 改善案
2. ...

#### 良い点
1. ...

#### 推奨 operation
- 次に適用すべき operation とその理由
```

チェック項目:
- 視覚階層（ユーザーの目が正しい順序で動くか）
- 情報密度（詰めすぎ or スカスカ）
- インタラクション明瞭性（クリッカブルが分かるか）
- ブランド一貫性（同じUIに見えるか）
- AI臭（anti-patterns.md の項目）
- a11y（コントラスト比、フォーカス順序、aria）
- パフォーマンス（不要なリペイント、重いアニメーション）

## energize — 活性化

目的: 機能的だが退屈なUIにエネルギーを注入する。

手法（状況に応じて選択）:
- **colorize**: アクセントカラーの導入・強化。モノクロ → 1色追加、2色 → コントラスト強化
- **animate**: ページロード時のスタッガードリビール、スクロールトリガー、ホバーのマイクロインタラクション
- **bolder**: フォントウェイト・サイズのコントラスト強化、ヒーロー要素の拡大、余白の大胆な使い方

注意:
- 3手法を全部適用しない。UIの弱点に応じて1-2手法を選ぶ
- animate は CSS-only を優先。React の場合は Motion ライブラリ
- 過剰にならないよう、適用後に calm の視点で確認する

## calm — 鎮静

目的: 過剰なUIを落ち着かせ、明瞭さを上げる。

チェックリスト:
- [ ] 彩度を下げる（特にアクセント色）
- [ ] アニメーションの duration を短縮 or 除去
- [ ] 情報密度を下げる（余白を増やす）
- [ ] フォントサイズの差を縮める（コントラスト過剰の場合）
- [ ] clarify: UXコピーの明確化（曖昧なラベル、不要な装飾テキスト）
- [ ] 装飾的グラデーション → ソリッドカラー or 控えめなグラデーションへ
