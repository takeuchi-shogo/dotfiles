# Timekeeper

朝の計画（plan）と夕方の振り返り（review）を対話で行い、Obsidian Vault に保存する。

## 設定

- **Vault パス**: `/Users/takeuchishougo/Documents/Obsidian Vault`
- **保存先**: `07-Daily/YYYY-MM-DD.md`
- **今日の日付**: コマンド実行時の日付を使用

## 引数の解釈

- `/timekeeper plan` → plan モード
- `/timekeeper midday` → midday check モード（正午の1問チェック）
- `/timekeeper review` → review モード
- `/timekeeper`（引数なし） → plan モード

---

## 共通ルール: 深掘り

各質問で最大2回まで深掘りする。基準は質問タイプごとに異なる。

### タスク系（plan Q1-Q3）

以下の3つを満たしたら次へ進む:
1. **具体的なアクション**（動詞+目的語）になっているか
2. **完了条件**が明確か（「〜したら終わり」が言える）
3. **時間の見積もり**が出ているか

深掘り例: 「具体的には何をしますか？」「何ができたら終わりですか？」「どれくらいかかりそう？」

### 制約系（plan Q4-Q5）

以下を満たしたら次へ進む:
- **具体性**のみ — 何を / 何に対して が明確か

深掘り例: 「具体的にはどういうことですか？」

### 内省系（review Q1-Q6）

以下を満たしたら次へ進む:
1. **具体性** — 抽象的すぎないか（「色々あった」→「何があった？」）
2. **因果** — 原因・理由が言語化されているか（「なぜそう思う？」）

深掘り例: 「もう少し具体的に教えてください」「なぜそう感じましたか？」

### 評価系（review Q7）

Q7 は構造化された3段階の質問（点数→理由→+1点）で深掘りが組み込まれているため、追加の深掘りは不要。

**重要**: 1メッセージにつき1質問だけ。複数の質問を一度に投げない。

## Plan モード

### Step 1: 前日ノートの確認

1. 前日の日付（カレンダー上の前日）で `07-Daily/YYYY-MM-DD.md` を Read で直接開く
2. **見つかった場合**: `### 明日に持ち越し` セクションを探し、内容があればユーザーに表示する
   - 「昨日の持ち越しがあります:（内容表示）。これも踏まえて今日の計画を立てましょう。」
   - セクションが存在しない / 空の場合はスキップ
3. **ファイルが存在しない場合**: スキップして Step 2 へ進む
   - 「では今日の計画を立てましょう。」

### Step 2: 対話（1問ずつ、順番に）

以下の質問を **1つずつ** ユーザーに聞く。各回答に対して深掘りルールを適用する。

**Q0 (Non-negotiables)**: 「今日、なりたい自分として **必ずやる3つの行動** は何ですか？（例: 朝の深い作業30分 / 運動 / 読書15分）」
- 既に確立している日次習慣があれば、それを確認するだけで OK（深掘り不要）
- 3つ揃わない場合は 1-2個で進めて良い（空でも可）
- **Anti-Pattern**: identity 宣言（"I am the type of person who..."）や人物評価に踏み込まない。単なる行動リスト確認に留める

**Q1**: 「今日の最重要タスクは何ですか？ これさえ終われば今日はOK、というものを1つだけ教えてください。」

**Q2**: 「今日やることを優先順に3〜5個挙げてください。」

**Q3**: Q2 で挙がったタスクを **1つずつ確認しながら** 時間ブロックを決める。
- 「では『{タスク1}』はいつやりますか？ 時間ブロックと見積もりを教えてください。（例: 9:00-10:30、約1.5h）」
- 全タスク分を順に聞く

**Q4**: 「今日あえて『やらない』と決めることは何ですか？ 意図的に捨てるものを宣言してください。」

**Q5**: 「邪魔されそうなものは何ですか？ それに対する予防策も一緒に教えてください。」

### Step 3: 確認と保存

1. 全回答を一覧で表示する
2. 「これで確定していいですか？ 修正したい項目があれば番号で教えてください。」と聞く
3. 修正があれば該当項目に戻る
4. 確定したら `07-Daily/YYYY-MM-DD.md` を保存する:
   - **ファイルが存在しない場合**: Write で新規作成
   - **ファイルに Plan セクションがない場合**: Edit で Plan セクションを追記
   - **ファイルに Plan セクションが既にある場合**: 「今日の計画は既に作成済みです。上書きしますか？」と確認し、yes の場合のみ Plan セクションを Edit で置換

```markdown
---
created: "YYYY-MM-DD"
tags:
  - type/daily
  - status/seed
---

# YYYY-MM-DD

## Plan

### Non-negotiables (今日必ずやる3つ)
- {Q0 の行動1}
- {Q0 の行動2}
- {Q0 の行動3}

### 最重要タスク
- {Q1の回答}

### やることリスト
1. {タスク} — {時間ブロック} — {見積もり}
2. ...

### やらないこと
- {Q4の回答}

### 邪魔への予防策
- {想定される邪魔} → {予防策}
```

5. 「保存しました: `07-Daily/YYYY-MM-DD.md`」と表示
6. 「今日も良い1日を！」で締める

## Midday モード

正午の autopilot 検出用。1問だけ聞いて当日ノートに追記する。**最大30秒で完結させること**。

### Step 1: 当日 Plan の確認

1. `07-Daily/YYYY-MM-DD.md` を Read で開く
2. **Plan セクションがある場合**: Non-negotiables と最重要タスクを抽出して表示
3. **Plan がない場合**: 「今日の計画がまだありません。`/timekeeper plan` を先に実行してください」で終了

### Step 2: 1問だけ質問

**Q**: 「午前中、朝の計画（最重要タスク: {task} / Non-negotiables）から逸脱していますか？ (はい / いいえ / 部分的)」

- 「いいえ」→ Step 3 へ（記録のみ）
- 「はい」or 「部分的」→ 追加1問: 「何に時間を使っていましたか？（事実のみ、1行）」
- **深掘り禁止**: 自己分析・感情・原因追求には踏み込まない。事実の1行記録のみ

### Step 3: 追記保存

`07-Daily/YYYY-MM-DD.md` に Midday セクションを Edit で追記（既にあれば上書き確認）:

```markdown
## Midday Check ({HH:MM})

- 逸脱: はい / いいえ / 部分的
- 実際に使っていた時間: {1行}
```

「記録しました。午後も良い時間を！」で締める。

## Review モード

### Step 1: 当日ノートの確認

1. `07-Daily/YYYY-MM-DD.md` を Read で読む
2. **Plan セクションがある場合**: 朝の計画を表示する
   - 「朝の計画を確認します:（内容表示）。これを踏まえて振り返りましょう。」
3. **ファイルがない / Plan がない場合**: そのまま振り返りを開始
   - 「今日の振り返りを始めましょう。」

### Step 2: 対話（1問ずつ、順番に）

**Q1**: 「今日やれたことは何ですか？」

**Q2**: 「やれなかったことは何ですか？」
- 回答が「なし」「全部やれた」等の場合: Q3 をスキップして Q4 へ

**Q3**: 「なぜやれなかったですか？ 原因を言語化してください。」

**Q4**: 「邪魔になったものはありましたか？（割り込み、通知、会議…）」

**Q5**: 「明日に持ち越すことはありますか？」

**Q6**: 「今日の学び・気づきはありますか？」

**Q7**: 「今日、考えや信念が変わったことはありますか？ 『こう思っていたけど、実はこうだった』のような気づきがあれば教えてください。」
- 回答が「なし」「特にない」等の場合: Q8 へ進む

**Q8**: 「まだ答えが出ていない問い、モヤモヤしていることはありますか？」
- 回答が「なし」の場合: Q9 へ進む

**Q9**: 「今日の自分に点数をつけるとしたら、まず何を基準にしますか？（例: 達成率、集中度、満足度…）」
- 基準が決まったら: 「では、その基準で何点ですか？」
- 回答後: 「なぜその点数ですか？」
- 回答後: 「何があれば+1点でしたか？」

### Step 3: 確認と保存

1. 全回答を一覧で表示する
2. 「これで確定していいですか？ 修正したい項目があれば番号で教えてください。」と聞く
3. 修正があれば該当項目に戻る
4. 確定したら `07-Daily/YYYY-MM-DD.md` に Review セクションを保存する:
   - **ファイルが存在し Review セクションがない場合**: Edit で末尾に追記
   - **ファイルが存在し Review セクションが既にある場合**: 「今日の振り返りは既にあります。上書きしますか？」と確認し、yes の場合のみ Edit で置換
   - **ファイルが存在しない場合**: frontmatter + Review セクションで Write する

```markdown

---

## Review

### やれたこと
- {Q1の回答}

### やれなかったこと
- {内容} — 原因: {Q3の回答}

### 邪魔になったもの
- {Q4の回答}

### 明日に持ち越し
- {Q5の回答}

### 今日の学び
- {Q6の回答}

### 信念の変化
- {Q7の回答}

### 未解決の問い
- {Q8の回答}

### 自己評価
- 基準: {Q9で決めた基準}
- 点数: {N点}
- 理由: {なぜその点数か}
- +1点にするには: {回答}
```

5. 「保存しました: `07-Daily/YYYY-MM-DD.md`」と表示
6. 「お疲れさまでした！」で締める

## Data Storage

計画・振り返りの履歴を `~/.claude/skill-data/timekeeper/` に蓄積します。
過去の計画精度（計画 vs 実績）を追跡し、見積もり精度の改善に活用します。

### 保存先
- `~/.claude/skill-data/timekeeper/sessions.jsonl` — 計画/振り返りの履歴 (append-only)

### フォーマット (1行1JSON)
```json
{"date": "2026-03-18", "type": "plan", "tasks_planned": 5, "focus": "harness改善"}
{"date": "2026-03-18", "type": "review", "tasks_completed": 3, "tasks_carried": 2, "learning": "見積もりが楽観的すぎた"}
```

### 使い方
1. 朝の計画完了後に plan エントリを追記
2. 夕方の振り返り完了後に review エントリを追記
3. 次回の計画時に過去の完了率を参照し、「前週の完了率: 60% → タスク数を調整しましょう」と提案

## Anti-Patterns

- **複数の質問を一度に聞かない** — 必ず1メッセージ1質問
- **ユーザーの回答を勝手にリライトしない** — 言語化したそのままの表現を保存する
- **深掘りを3回以上しない** — 最大2回で次へ進む
- **plan の項目が空のまま先に進まない** — 全項目が埋まるまで続ける
- **review で plan の達成/未達成を勝手に判定しない** — ユーザー自身に振り返らせる
- **既存セクションを無断で上書きしない** — plan/review が既にある場合は必ず確認する
