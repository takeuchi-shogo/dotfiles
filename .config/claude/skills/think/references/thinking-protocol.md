# Thinking Protocol

AI を「答えを出す機械」ではなく「思考を研ぐパートナー」として使うための3ルール。

## 3 Rules

### Rule 1: 先に自分の考えを述べる

回答を求める前に、自分の現在の考えを述べる。半熟でも、間違っていてもいい。

- NG: 「これどうすればいい？」（思考の外注）
- OK: 「自分はこう考えている。理由は〜。ここに自信がない。」（思考の研磨）

**なぜ**: 自分の考えを言語化する行為そのものが思考を整理する。AI はゼロから答えを出すより、既存の考えの穴を突く方が遥かに有用。

### Rule 2: 支持より反論を先に求める

自分のポジションへの最強の反論を求めてから、支持を求める。

- NG: 「この方針でいいよね？」（確認バイアスの強化）
- OK: 「この方針の最大の弱点は何？ 一番強い反論をくれ。」（思考の耐久テスト）

**なぜ**: AI は sycophancy（追従バイアス）を持つ。明示的に反論を求めないと、ユーザーの考えを補強する方向に偏る。

### Rule 3: 「何が見えていないか」で締める

セッションの最後に必ず問う: **What am I not seeing?**

**なぜ**: 自分の脳は問題に近すぎて盲点を検出できない。利害関係のない思考パートナーだけが、避けている問いを指摘できる。

## Session Structure

```
1. CONTEXT    — 思考コンテキストドキュメントをロード
2. THEME      — 今日考えたいテーマを1文で述べる
3. POSITION   — 現在の考えとその理由（Rule 1）
4. CHALLENGE  — AI が反論・盲点を突く（Rule 2）
5. DIALOGUE   — 深掘り対話（必要なだけ）
6. BLIND SPOT — 「何が見えていないか」（Rule 3）
7. CAPTURE    — 思考の変化・新しい問いを記録
```

## Anti-Patterns

| NG | 理由 |
|----|------|
| AI に結論を出させる | 思考の外注。自分で決める力が退化する |
| 自分の考えなしに質問する | AI は方向性がないと汎用的な回答しかできない |
| 心地よい回答で満足する | 確認バイアスの罠。不快な反論にこそ価値がある |
| 毎回新しいセッションで始める | コンテキストの断絶。積み上がらない |
