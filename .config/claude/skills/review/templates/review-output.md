# Review Output Template

レビュー結果の統合出力フォーマット。Step 4 (Synthesis) で使用する。

---

## 出力フォーマット

```markdown
## レビュー結果

**変更概要**: {変更行数}行 / {ファイル数}ファイル / {言語リスト}
**レビューアー**: {起動したレビューアーのリスト}

### Critical（修正必須）

- **[{レビューアー名}]** `{file}:{line}` — {指摘内容}
  → {推奨修正}

### Important（検討推奨）

- **[{レビューアー名}]** `{file}:{line}` — {指摘内容}
  → {推奨修正}

### Suggestion（参考）

- **[{レビューアー名}]** `{file}:{line}` — {指摘内容}

### 判定

{PASS / NEEDS_FIX / BLOCK}

- **PASS**: 指摘なし、または Suggestion のみ。次のステップへ進んでよい
- **NEEDS_FIX**: Important 以上の指摘あり。修正後に再レビュー推奨
- **BLOCK**: Critical な指摘あり。修正必須
```

## 統合ルール

### 重複排除

同じファイル・行への指摘が複数レビューアーから上がった場合:

1. 最も具体的な指摘を残す（「問題がある」より「NullPointerException の可能性」）
2. 重要度が異なる場合は高い方を採用
3. 排除された指摘は「(他 N 件のレビューアーも同様の指摘)」と注記

### 重要度マッピング

各レビューアーの出力を統一的な重要度にマッピングする:

| レビューアーの表現               | 統合後の重要度 |
| -------------------------------- | -------------- |
| MUST, Critical, CRITICAL, 90-100 | **Critical**   |
| CONSIDER, Important, HIGH, 80-89 | **Important**  |
| NIT, Suggestion, MEDIUM, Minor   | **Suggestion** |

### 指摘がない場合

```markdown
## レビュー結果

**変更概要**: {変更行数}行 / {ファイル数}ファイル / {言語リスト}
**レビューアー**: {起動したレビューアーのリスト}

指摘なし。

### 判定

PASS
```
