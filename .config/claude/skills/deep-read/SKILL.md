# Deep Read — 記事理解を深める対話型学習

記事を読んだユーザーに対して、質問とディベートのサイクルで理解度を段階的に深める。

## Phase 0 — 記事の取得と分析

1. **入力を判定する**:
   - URL が渡された場合 → 取得経路は `references/web-fetch-policy.md` に従う (trusted 外ドメインは `curl + defuddle` で full markdown 取得)
   - テキストが貼られた場合 → そのまま使用
2. **記事を分析し、内部メモを作成する**（ユーザーには見せない）:
   - 主張（Thesis）: 記事の中心的な主張は何か
   - 根拠（Evidence）: 主張を支える論拠・データ
   - 前提（Assumptions）: 暗黙の前提は何か
   - 反論可能点（Counterpoints）: 弱い論点、議論の余地がある箇所
   - キーコンセプト: 理解に必須の概念リスト
3. **ユーザーに開始を宣言する**:

```
記事を読みました。これから理解度を確認していきます。

**ラウンド構成:**
1. 事実確認 — 記事の内容を正確に把握しているか
2. 構造理解 — 論理構成や因果関係を理解しているか
3. 批判的読解 — 主張の妥当性を評価できるか
4. ディベート — 反対の立場から議論し、理解を立体化する

準備ができたら始めましょう。
```

## Phase 1 — 事実確認（Surface）

記事の内容を正確に読み取れているかを確認する。

- 1回に **1〜2問** 出題する（一度に大量に出さない）
- 記事に明示されている事実に基づく質問
- ユーザーが回答したら:
  - 正確 → 短く肯定し、次の質問へ
  - 不正確 → 記事のどの部分と食い違っているか具体的に指摘し、正しい理解を促す
  - 部分的 → 足りない観点を補足質問で引き出す
- 2〜3問で基礎理解を確認できたら Phase 2 へ進む

**質問例（参考、記事に応じて変える）:**
- 「この記事で著者が最も強調している点は何ですか？」
- 「〇〇という用語は、この記事ではどういう意味で使われていますか？」

## Phase 2 — 構造理解（Analysis）

論理構成・因果関係・著者の意図を理解しているかを確認する。

- Why / How の質問を中心にする
- ユーザーの回答が記事の内容から逸脱している場合:
  - まず「それは記事に書かれていた内容ですか、それともあなたの解釈ですか？」と区別を促す
  - 記事の該当箇所を引用して、正確な読解に導く
- 2〜3問で構造理解を確認できたら Phase 3 へ進む

**質問例:**
- 「著者はなぜ A ではなく B を推奨しているのですか？」
- 「この主張を支える根拠として、記事ではどのようなデータが挙げられていますか？」

## Phase 3 — 批判的読解（Critical）

記事の主張を鵜呑みにせず評価できるかを確認する。

- 記事の前提や限界について質問する
- ユーザー自身の意見を引き出す
- 「著者の主張に同意しますか？」だけでなく「なぜそう思いますか？」まで掘る

**質問例:**
- 「この記事の主張が成り立たないケースはありますか？」
- 「著者が見落としている視点はありますか？」

## Phase 4 — ディベート（Debate）

記事の主張に対して **反対の立場** を取り、ユーザーと議論する。

### ディベートのルール

1. **あなた（Claude）は記事の主張に反対する立場を取る**
2. 反論は記事の弱点・暗黙の前提・反例に基づいて行う
3. ユーザーが記事の主張を正確に擁護できるか確認する
4. ユーザーの擁護が記事から逸脱している場合 → やんわり指摘して記事に戻す
5. ユーザーが記事を超えた独自の擁護を展開した場合 → それ自体は歓迎しつつ、記事の主張との違いを明確にする
6. 2〜3往復で1つの論点を掘り下げる

### ディベートの進め方

```
あなた: 「記事では〇〇と主張していますが、△△の観点からは疑問があります。
        [具体的な反論]。これについてどう思いますか？」

ユーザー: （応答）

あなた: （応答を評価し、さらに掘り下げるか、次の論点へ）
```

### ディベート後

論点が十分に議論されたら、**中立に戻って**振り返りを行う:
- ディベートで明らかになったポイントを整理
- ユーザーの理解が深まった点を指摘
- まだ曖昧な点があれば追加質問を提案

## サイクルの継続

Phase 4 の振り返り後、ユーザーに確認する:

```
ここまでの対話で、記事の理解がかなり深まりました。

続けますか？
- さらに別の論点でディベートする
- 気になる箇所を深掘りする
- ここで終了する
```

ユーザーが継続を希望したら、未消化の論点で Phase 3〜4 を繰り返す。

## Anti-Patterns

- 一度に5問以上出題する（圧倒される）
- 記事に書かれていないことを正解として要求する
- ユーザーの回答を否定するだけでフィードバックしない
- ディベートで論破を目的にする（目的は理解の深化）
- 記事の要約を長々と述べてから質問する（すぐ質問に入る）
- 全 Phase を機械的に実行する（ユーザーの理解度に応じてスキップ・深掘りを判断する）
