# Profile Drip — 1日1問プロファイル構築

既存の type:user メモリのギャップを見つけ、1問だけ質問してプロファイルを漸進的に充実させる。

## 手順

1. `memory/user_*.md` を全て読む
2. developer-onboarding スキルのカバー領域と比較してギャップを特定
3. ギャップから最も価値のある1つを選び、自然な対話として質問
4. 回答を受けたら該当メモリファイルを更新（なければ新規作成）

## ルール

- 1回の実行で質問は **1問だけ**
- サーベイ口調禁止。「ところで、〜って普段どうしてますか？」のような自然な形式
- 回答が短くても深追いしない
- 既に回答済みの内容を再質問しない

## スキルギャップ分析（記事 #20 Skill Gap Analyzer 統合）

ギャップを検出した際、単に質問するだけでなく以下の観点も簡潔に提示する:

1. **最短パス**: そのスキルを最速で身につける方法（推奨リソース1つ）
2. **スキップ可能**: 目標達成に影響しないなら「これは後回しでOK」と明示
3. **最大レバレッジ**: 「これを覚えると他の3つも楽になる」のような複合効果がある場合に指摘

### 適用ルール
- 1問ルールは維持。ギャップ分析は質問の補足として簡潔に（3行以内）
- 過度な学習計画は提示しない。次の1歩だけ
- ユーザーの現在のスタック（Go/TypeScript/Rust学習中）を考慮
