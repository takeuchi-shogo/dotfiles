## 手順

1. `.tmp/implement-feedback.md` が存在すれば読み込み、前回の指摘に対応する
2. 初回はタスクを実装する。2回目以降は指摘された箇所を修正する
3. テストを実行し、全テストが pass することを確認する
4. 変更内容をセルフレビューする:
   - `git diff --stat HEAD` で変更規模を確認
   - 変更ファイルごとに以下を検証:
     a. 正確性: ロジックの正しさ、エッジケース
     b. スタイル: 命名、フォーマット、言語の慣例
     c. セキュリティ: インジェクション、認証漏れ
     d. 影響範囲: 変更が他ファイルに与える影響
5. レビュー結果を `.tmp/implement-feedback.md` に書き出す:
   - PASS: 全項目問題なし → `<promise>REVIEW_COMPLETE</promise>` を出力
   - NEEDS_FIX: 指摘あり → 具体的な指摘内容をファイルに書き、次のイテレーションで修正
6. NEEDS_FIX の場合はステップ 1 に戻る

## レビュー基準

- テストが全て pass すること
- 新しい lint 警告がないこと
- セキュリティ上の問題がないこと
- コードの意図が明確であること
- 不要な変更が含まれていないこと

## 重要

- 各イテレーションで修正した内容を簡潔にログに残す
- 同じ指摘を2回以上繰り返す場合はアプローチを変える
- テストが存在しない場合は、修正箇所のテストを追加する
