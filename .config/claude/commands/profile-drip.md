# Profile Drip — 1日1問プロファイル構築

既存の type:user メモリのギャップを見つけ、1問だけ質問してプロファイルを漸進的に充実させる。

## 手順

1. `memory/user_*.md` を全て読む
2. developer-onboarding スキルのカバー領域と比較してギャップを特定
3. ギャップから最も価値のある1つを選び、自然な対話として質問
4. 回答を受けたら該当メモリファイルを更新（なければ新規作成）

## ルール

- 1回の実行で質問は **1問だけ**
- サーベイ口調禁止。「ところで、〜って普段どうしてますか？」のような自然な形式
- 回答が短くても深追いしない
- 既に回答済みの内容を再質問しない
